# SQL 기초 배워보기

수많은 데이터를 텍스트나 엑셀 파일로 관리할 경우 단점
1) 매번 파일을 전송하는 번거로움
2) 대량의 데이터를 다루기가 힘듦
3) 파일 삭제, 하드 디스크 고장, 보안 문제 등 사고에 대응하기 어려움
-> 이러한 이유로 실무에서는 대부분 데이터 베이스를 이용해 데이터를 효율적으로 관리

DB란 여러 사람이 공유해 사용할 목적으로 체계화하여 통합, 관리하는 데이터의 집합

투자와 관련된 모든 데이터를 데이터베이스에 저장한 후 이를 관리하거나 사용 가능
데이터베이스 관리 시스템 (DBMS)란 다수의 사용자들이 DB 내의 데이터를 접근할 수 있도록 해주는 SW 도구의 집합
DBMS는 대량의 데이터를 다수의 사람이 안전하고 간단히 다룰 수 있게 해주는 장점

관계형 데이터베이스 (Relational DB): DBMS 중 가장 일반적으로 사용되는 건
열과 행의 2차원 표 형식으로 데이터를 관리하거나 이해하기 쉬움
SQL이라는 전용 언어를 사용해 데이터 처리 가능
관계형 DBMS 중 대표적으로 사용 되는 것: 오라클 RDBMS, MS SQL server, Postgre SQL, MySQL 등

DB: 각각의 테이블로 구성
열(컬럼): 테이블에 보관하는 데이터 항목
행(레코드): 데이터 한 건에 해당하며, RDBMS는 반드시 행 단위로 데이터를 읽고 씀
셀(값): 행과 열이 교차하는 값이며, 하나의 셀 안에는 하나의 데이터만 넣을 수 있음
